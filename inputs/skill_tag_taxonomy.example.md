# Skill Tag Taxonomy

experience_curator가 자산 문서에 역량 태그를 부여할 때 참조하는 분류 체계.
원본 경험에 명시된 기술/역할만 태그로 부여하며, 이 목록에 없는 태그도 필요 시 추가 가능하다.

## 1. 기술 역량 (Technical)

### 프론트엔드
`#React` `#Vue.js` `#JavaScript` `#RealGrid2`

### 백엔드
`#Java` `#Python` `#FastAPI` `#Maven` `#SQL` `#MSSQL`

### AI/LLM
`#LangChain` `#LLM프롬프트엔지니어링` `#DuckDB` `#온톨로지설계`

### 인프라/연계
`#EAI` `#pom.xml의존성관리`

## 2. 도메인 역량 (Domain)

`#SCM` `#APS` `#생산계획` `#공급망` `#제조운영` `#KD_Planning` `#재고관리`

## 3. 역할 역량 (Role)

`#UI개발` `#시스템설계` `#엔진연계` `#도메인구조화` `#AI응용설계` `#백엔드공통구조` `#데이터모델링`

## 4. 소프트 역량 (Soft)

`#협업` `#요구사항분석` `#커뮤니케이션` `#도메인전문가협업` `#사용자피드백반영`

## 사용 규칙

- 각 자산 파일에 최소 2개 분류에서 태그 부여
- 원본에 없는 역량은 태그로 부여하지 않음
- 맥락상 명백한 경우에만 `(추정)` 주석과 함께 부여 가능
- 새 태그가 필요하면 이 파일에 먼저 추가 후 사용

# User Config (Example)

> 이 파일은 예시입니다. `user_config.md`로 복사한 뒤 개인 설정으로 수정하세요.
> `user_config.md`는 .gitignore에 의해 커밋되지 않습니다.

사용자 개인 설정. agent들이 JD 필터링, 이력서 작성, 지원 전략 수립 시 참조한다.

## 경로 설정

> 세션 시작 시 CLAUD.md가 이 값을 읽어 모든 파일 경로를 resolve한다.
> `[CAREER_DIR]` / `[PROJECTS_DIR]` 플레이스홀더가 아래 절대 경로로 치환된다.

```
career_dir: /path/to/your/Career
projects_dir: /path/to/your/Projects
```

- `career_dir`: 경험 데이터가 담긴 Career 폴더 절대 경로
  예: `/Users/yourname/Obsidian/Career` 또는 `C:/Users/yourname/Obsidian/Career`
- `projects_dir`: 작업 로그·산출물이 담긴 Projects 폴더 절대 경로
  예: `/Users/yourname/Obsidian/Projects`

## 희망 근무 지역

| 순위 | 지역 | 비고 |
|------|------|------|
| 1순위 | [주 생활권 지역] | 주 생활권 |
| 2순위 | [2순위 지역] | |
| 3순위 | [3순위 지역] | 최후의 보루 |

- 위 지역 외 공고는 **수집·분석 대상에서 제외**한다.
- 원격근무(재택) 공고도 지역 조건 예외 없이 동일하게 적용한다 (원격은 고려하지 않음).

## 기업 형태

| 순위 | 기업 형태 |
|------|-----------|
| 1순위 | 대기업 |
| 2순위 | 중견기업 |
| 3순위 | 외국계 |
| 4순위 | 대기업(중견기업) 자회사 |
| 5순위 | 강소기업 |

- 기업 형태 순위(1~5순위) 기준으로 우선 수집한다.
- 5순위(강소기업)이거나 기업 형태 분류 불명확한 경우 → 수집하되 "기업형태 미달" 표시.
- 지역 일치 공고가 희소한 경우, 기업 형태 1~4순위에 해당하는 공고에 한해 전국 범위로 확장 수집할 수 있다.
